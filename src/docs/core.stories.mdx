# Design Tokens

Design tokens are design decisions, represented as data, that ensure systematically unified and cohesive product experiences.

## What are design tokens?

Design tokens represent the whole set of values involve in building and maintaining a design system such as:

- Spacing,
- Colour,
- Typography,
- Object styles,
- Animation

And many more.

They are used in place of hard-coded values in order to ensure flexibility and unity across all product experiences.

Design tokens are integrated to our components and cover colour themes, component states, etc.

## Design Token Types

The following types of design tokens serve as the building blocks which make up the Light Design Language.

### Global Tokens (Core)

Global tokens are the primitive values of the design language and are represented by context-agnostic names. The colour palette, animations, typography and spacing value are all global tokens.

They can be used directly or by transitivity through context composition via the other token types.

### Component-specific Tokens

Component-specific tokens are an exhaustive representation of every value associated with a component. They can inherit from global tokens.

The main difference with global tokens is that component-specific tokens are named in a way that allows engineering teams to be as specific as possible in applying tokens in component development.
